Техническое задание на разработку сервиса «Check Site»

Версия: 2.0
Дата: Октябрь 2025

================================================================================

1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

1.1 Назначение
Check Site — веб-сервис для контроля качества строительных работ, предназначенный для строительно-монтажных компаний (СМК).

1.2 Основная цель
Обеспечить прозрачный контроль устранения замечаний на строительных объектах через систему фотофиксации "До → После → Осмотр".

1.3 Ключевые преимущества
- Простая трехэтапная фиксация замечаний
- Мгновенные уведомления подрядчикам через Telegram
- Наглядный контроль сроков исполнения
- Минимальное время на обучение персонала

================================================================================

2. СТРУКТУРА ПОЛЬЗОВАТЕЛЕЙ И РОЛИ

2.1 Иерархия должностей в СМК

РУКОВОДСТВО:
• Генеральный директор
• Технический директор
• Финансовый директор
• Главный инженер
• Начальник ПТО
• Менеджер проекта
Роль в системе: Просмотр и контроль

ИТР:
• Инженеры ПТО
• Руководитель проектов
• Начальник участка
• Прораб
• Мастер
Роль в системе: Создание и управление замечаниями

КОНТРОЛЬ:
• Технадзор
• Авторский надзор
Роль в системе: Создание и проверка замечаний

ИСПОЛНИТЕЛИ:
• Бригадир
• Подрядчик
Роль в системе: Устранение замечаний

2.2 Права доступа по ролям

РУКОВОДСТВО:
• Просмотр всех замечаний по выбранным объектам
• Доступ к отчетам и статистике
• Управление пользователями и объектами

ИТР/КОНТРОЛЬ:
• Создание замечаний (этап "До")
• Редактирование замечаний
• Назначение подрядчика
• Проверка выполнения (этап "Осмотр")
• Установка сроков и технологии исправления

ПОДРЯДЧИК:
• Просмотр назначенных замечаний
• Отметка о выполнении (этап "После")
• Загрузка фото выполненных работ
• Добавление комментариев

3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

3.1 Регистрация и авторизация

Первичная регистрация (только для Руководства):

Поля регистрации:
- Основной email (логин)
- Запасной email
- ФИО
- Должность (выбор из списка)
- Пароль и подтверждение

После регистрации:
- Автоматическая отправка данных входа на email
- Автоматический вход в систему

Создание пользователей:
- Руководство создает учетные записи для всех категорий
- Автоматическая генерация паролей
- Отправка данных для входа на email пользователя

3.2 Управление строительными объектами

Поля объекта:
- Название объекта
- Адрес
- Заказчик
- Сроки строительства
- Ответственные лица

Привязка пользователей:
- Назначение ИТР на объекты
- Назначение подрядчиков
- Установка прав доступа

3.3 Система замечаний "До → После → Осмотр"

ЭТАП "ДО" (создание замечания)
Заполняет: ИТР или Контроль

Обязательные поля:
- Название замечания
- Описание проблемы
- Объект и участок
- Фото дефекта (опционально)
- Назначенный подрядчик
- Срок устранения
- Технология исправления

Приоритеты:
- Критичное (влияет на безопасность)
- Важное (влияет на качество)
- Обычное (косметические дефекты)

ЭТАП "ПОСЛЕ" (устранение)
Заполняет: Подрядчик

Действия:
- Отметка "Выполнено"
- Загрузка фото выполненных работ
- Комментарий о проделанной работе
- Указание использованных материалов (опционально)

ЭТАП "ОСМОТР" (проверка)
Заполняет: ИТР или Контроль

Варианты решений:
- Принято - работа выполнена качественно
- Доработать - требуется доработка (новый срок)
- Отклонено - переделать заново

3.4 Статусы замечаний

- Новое: Замечание создано, ожидает начала работ (серый)
- В процессе: Подрядчик приступил к устранению (синий)
- На проверке: Подрядчик выполнил, ожидает осмотра (желтый)
- Исполнено: Замечание принято и закрыто (зеленый)
- Просрочено: Истек срок устранения (красный)

3.5 Система уведомлений

Telegram-бот для уведомлений:
- Мгновенные Push-уведомления подрядчикам о новых замечаниях
- Напоминания о приближающихся сроках
- Уведомления об изменении статуса

Типы уведомлений:
- Новое замечание назначено
- Срок истекает через 24 часа
- Замечание просрочено
- Замечание принято
- Требуется доработка

4. ИНТЕРФЕЙС СИСТЕМЫ

4.1 Главная страница (Dashboard)

Виджеты для Руководства:
- Общее количество замечаний по объектам
- График: новые/закрытые за неделю
- Топ-5 подрядчиков по просрочкам
- Прогресс по объектам (%)

Виджеты для ИТР/Контроль:
- Мои активные замечания
- Требуют проверки
- Просроченные замечания
- Быстрое создание замечания

Виджеты для Подрядчика:
- Назначенные замечания
- Срочные (истекает сегодня)
- На доработке
- Выполненные за неделю

4.2 Мобильная версия

Приоритет на мобильных устройствах:
- Крупные кнопки для работы в перчатках
- Быстрая камера для фото
- Оффлайн режим для плохой связи на объекте
- Упрощенная навигация

================================================================================

5. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

5.1 Архитектура

Frontend:
- React + TypeScript
- Адаптивная верстка (mobile-first)
- PWA для работы оффлайн
- Простой и понятный UI

Backend:
- Node.js + Express/NestJS
- PostgreSQL
- JWT авторизация
- REST API

Интеграции:
- Telegram Bot API
- Email сервис (SendGrid/SMTP)
- Файловое хранилище (S3/локальное)

5.2 Требования к производительности
- Загрузка страницы < 3 сек
- Загрузка фото < 5 сек (с сжатием)
- Поддержка 100+ одновременных пользователей
- Резервное копирование ежедневно

5.3 Безопасность
- HTTPS обязательно
- Хеширование паролей (bcrypt)
- Защита от брутфорса
- Логирование всех действий

================================================================================

6. ОТЧЕТНОСТЬ

6.1 Виды отчетов
- Сводка по объекту - все замечания с фото
- Отчет по подрядчику - статистика выполнения
- Просроченные замечания - с указанием ответственных
- Акт выполненных работ - с фото "До/После"

6.2 Форматы экспорта
- PDF для печати актов
- Excel для анализа
- Прямая печать с мобильного

================================================================================

7. ЭТАПЫ ВНЕДРЕНИЯ

Этап 1: MVP (1 месяц)
- Регистрация и авторизация
- Создание объектов и пользователей
- Базовые замечания с фото
- Простые статусы

Этап 2: Основной функционал (1 месяц)
- Система "До-После-Осмотр"
- Telegram-уведомления
- Контроль сроков
- Базовые отчеты

Этап 3: Оптимизация (2 недели)
- Мобильная версия
- Оффлайн режим
- Массовые операции
- Расширенные отчеты

================================================================================

8. КРИТЕРИИ УСПЕШНОГО ЗАПУСКА

- Регистрация занимает < 2 минут
- Создание замечания < 1 минуты
- Уведомление приходит мгновенно
- Фото грузятся с телефона без проблем
- Отчеты формируются корректно
- Система работает на всех устройствах
